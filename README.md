# Repo Onboard

Repo Onboard scans a local repository and creates a Markdown onboarding report
that helps a developer understand the project structure, likely entry points,
tooling, startup paths, import relationships, and open questions.
The LLM-backed report includes a `Projektstruktur` section that explains each
relevant file and folder for teammates who are new to the repository.

It also extracts common onboarding signals:

- all `package.json` scripts
- Makefile targets with short recipe previews
- lightweight Python import relationships
- architecture roles such as `api/`, `routes/`, `models/`, `services/`,
  `components/`, `pages/`, and `migrations/`

## Quick Start

```bash
python -m onboard scan /path/to/repo
```

You can also scan a GitHub repository URL directly:

```bash
python -m onboard scan https://github.com/MoAlj/Onboard.git
```

Remote URLs are cloned into a temporary directory with `git clone --depth 1`,
scanned, and removed afterwards.

After installing the package, the console script is available too:

```bash
onboard scan /path/to/repo --output onboarding.md
```

Useful scan options:

```bash
onboard scan /path/to/repo --max-files 1000 --snippet-bytes 4000
```

- `--max-files` limits how many files are inspected.
- `--snippet-bytes` limits how many bytes are read from each relevant text file.
- `--no-llm` disables the LLM and prints a reduced debug report.
- `--verbose` prints scan details and optional LLM provider warnings.

The scanner only reads curated files such as READMEs, project manifests,
Docker/Make files, CI files, and likely entry points. Secret-like files such as
`.env` and private keys are skipped.

Normal scans require `GEMINI_API_KEY`, `GOOGLE_API_KEY`, or `OPENAI_API_KEY`.
The deterministic scanner still collects facts locally, but the startup guide
and file-by-file project structure explanation are generated by the configured
LLM. Use `--no-llm` only for offline debugging or tests.

## Gemini Free Tier Test

1. Create an API key in Google AI Studio.
2. Export the key:

```bash
export GEMINI_API_KEY="your-key"
```

3. Run a scan with the Gemini-backed report sections:

```bash
PYTHONPATH=src python -m onboard scan /path/to/repo
```

The default Gemini model is `gemini-3.5-flash`. You can override it:

```bash
export ONBOARD_GEMINI_MODEL="gemini-3.5-flash"
```

If Gemini takes longer on larger repositories, increase the request timeout:

```bash
export ONBOARD_GEMINI_TIMEOUT="180"
```
